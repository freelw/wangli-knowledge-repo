# Browser 专项维度每周跟进流程

## 适用场景

本流程用于每周生成 Browser 项目专项维度跟进文档。

目标不是写长报告，而是把 Notion 中分散的专项状态、周报、晨报、会议纪要、技术方案、发布记录，重新汇总回稳定的专项里程碑结构，让读者快速判断：

- 每个专项当前处于哪个里程碑。
- 本周哪些里程碑或子点有推进。
- 哪些里程碑已结束、正在进行、尚未开始。
- 哪些里程碑或子点出现风险、延期、阻塞。
- 本周变化相对上周发生在哪里。

## 强制原则

- prompt 来源必须是 Notion。
- 数据来源必须是 Notion。
- 不使用 Slock 讨论、PR 页面、knowledge-repo、本地记忆作为事实来源。
- Slock 只能作为用户指令来源，不能作为周跟进事实依据。
- 每条本周进展必须附 Notion 来源文档。
- 找不到 Notion 来源时，不写成确定结论，标记为 `来源待补充`。

## 搜索范围

通过 `Browser 文档导读（先看我）` 进入本周相关资料。

搜索范围包括：

- Browser 周报中心。
- Browser 晨报中心。
- 相关会议纪要。
- 相关需求文档。
- 相关技术方案。
- Notion 中的 PR / issue 汇总。
- Notion 中的发布记录。
- 专项子文档及其下一级跟踪文档。

搜索线索应以 `专项 + 当前里程碑 + 跟踪子点` 为主，不要只按日期粗扫。

## 输出文档

每周生成两个 Notion 文档：

```text
Browser 内核专项本周进展 - YYYY-MM-DD
browser 云平台专项本周进展 - YYYY-MM-DD
```

输出必须创建或更新为 Notion 文档，并放在 `Browser 专项维度` 下。

历史周进展不覆盖。新的本周进展只维护当前周。确需修历史时，只做事实修正，不追加新周进展。

## 标准流程

### 1. 读取专项结构

先读 `Browser 内核专项信息` 和 `browser 云平台专项信息`，提取：

- 当前有哪些专项。
- 每个专项负责人。
- 每个专项当前阶段。
- 每个专项全部里程碑。
- 正在进行的里程碑下的跟踪子点。

负责人、专项名、里程碑名、子点名以 Notion 专项信息页为准，不在周跟进文档中自行重命名。

### 2. 拉取本周事实

按专项逐项查找本周资料：

- 先看最近一篇 Browser 周报。
- 再看本周 Browser 晨报。
- 再看专项页和子文档中的本周更新。
- 必要时补充会议纪要、技术方案、发布记录、PR / issue 汇总。

只采用 Notion 中能看到的事实。

### 3. 匹配里程碑和子点

对每个专项判断：

- 本周进展落在哪个里程碑。
- 本周进展对应哪个子点。
- 是否改变了里程碑或子点状态。
- 是否出现风险、延期、阻塞。

进展不能脱离专项结构单独写。

### 4. 生成稳定结构

每个输出文档按专项展开。

每个专项必须列出全部里程碑，不能只列有进展的里程碑。

正在进行的里程碑必须展开子点。已结束或未开始的里程碑可以折叠成一行，避免文档膨胀。

### 5. 标记状态

统一使用以下状态：

- `✅ 已结束`
- `🟡 进行中`
- `⚪ 未开始`
- `🔴 风险 / 延误`

里程碑层级必须标状态。正在进行的里程碑下，跟踪子点也必须标状态。

## 输出模板

严格遵循该结构，不额外增加 TL;DR、来源说明、过程说明等段落。

```text
## 专项 1：专项名称
**负责人**：姓名
**当前阶段**：Mx 里程碑名称
### 里程碑进度
- `✅ M1 xxx`：已结束。
- `🟡 M2 xxx`：进行中。
	- `🟡 子点 A`：本周进展一句话。来源：Notion 文档链接
	- `⚪ 子点 B`：本周无新增。来源：Notion 文档链接
- `⚪ M3 xxx`：未开始。
```

## 写作规则

- 不写大段叙事。
- 不复制周报或晨报原文。
- 不把历史背景写成本周进展。
- 不把本周无来源的信息写成确定结论。
- 摘要尽量短，一条子点只写一句话。
- 文档结构每周保持稳定，方便人工对比。
- 没有新增进展的进行中子点，可以写 `本周无新增`，但仍要给出来源。

## 质量红线

以下情况必须返工：

- 只写本周有进展的专项，遗漏无进展专项。
- 没有列出全部里程碑。
- 正在进行的里程碑没有展开子点。
- 有进展的子点没有 Notion 来源。
- 使用了 Slock、PR、knowledge-repo、本地记忆作为事实来源。
- 把长期背景写成本周进展。
- 状态标记不一致。
- 输出文档增加额外说明段，导致模板不稳定。
- 文档随时间持续膨胀，无法快速对比。

## 2026-05-28 执行经验

本次执行中，规则 prompt 曾中途更新。正确做法是：

1. 重新读取 `每周跟进规则-prompt`。
2. 丢弃旧模板中额外的 TL;DR、prompt source、data source 说明段。
3. 重新读取专项主文档、专项子文档、周报、晨报和相关 Notion 记录。
4. 覆盖更新当周两个周跟进 Notion 页面。
5. 更新后再 fetch 两个页面，确认内容和父路径仍正确。

## 常见误区

- 不要把“资料来源来自 Notion”写进最终周跟进文档；这是执行约束，不是输出内容。
- 不要把 prompt 内容粘贴到输出文档。
- 不要因为某个专项本周无进展就省略该专项。
- 不要根据记忆推断本周状态，必须回到 Notion 来源。
- 不要把“目标日期存在”误写成“本周已完成”。
- 不要把当前阶段和里程碑状态混用：当前阶段来自专项页，状态来自本周事实判断。
