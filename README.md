# 后端知识仓库

## 作用

这个仓库用于沉淀后端平台的长期知识，和按日期堆积的任务记录分开管理。

这里主要区分 4 类内容：

1. 稳定知识
2. 运维操作
3. 架构设计
4. 任务归档

## 目录结构

- `00-overview`
  - 系统地图、仓库地图、环境说明、工程约束
- `01-architecture`
  - 核心架构与服务边界文档
- `02-operations`
  - 发布 runbook 与运维检查清单
- `03-infra`
  - 网络、镜像同步、基础设施专题
- `04-sdk`
  - SDK 全景、对齐记录与接入知识
- `05-domains`
  - 邮件送达率、POC SOP 等专项主题
- `90-task-archive`
  - 任务摘要与历史归档

## 编写规则

- 稳定、可复用的知识放在主题目录下。
- 一次性交付记录放在 `90-task-archive`。
- 不要在主知识层重新堆一套按日期分类的文档。
