# 代理服务法律咨询问题描述

## 一、咨询背景

我们公司计划/已经向用户提供一项代理网络能力，主要用于用户在使用我们的浏览器自动化、网页访问、网页信息抓取等功能时，按地域合规访问公开网页信息。

该能力不是 VPN、翻墙工具、匿名代理或规避网络监管工具。我们希望咨询律师：在当前技术限制和业务定位下，向用户提供这样的代理访问能力，以及用户通过该能力进行网页信息抓取时，是否存在法律风险；同时，我们需要让用户同意哪些条款、采取哪些产品和风控措施，才能尽量降低平台法律风险。

## 二、当前代理能力设计

### 2.1 代理 IP 池按地域隔离

当前代理 IP 池已经按地域隔离使用：

- 官方代理访问中国大陆网站时，只使用中国大陆 IP 代理。
- 香港区域或海外访问场景，使用海外 IP 代理。

### 2.2 不提供通过代理翻墙的能力

我们不会向用户提供“通过代理翻墙”的能力：

- 国内用户不能通过我们的代理服务访问依法不能访问的境外网站或服务。
- 系统侧已经通过第三方代理供应商的账号体系、区域配置、线路限制和我们自己的产品配置来约束代理出口。
- 我们的产品定位不是 VPN、翻墙工具、匿名代理或规避网络监管工具。

### 2.3 用户使用场景

用户使用代理能力的主要场景包括：

- 通过我们的浏览器云平台访问网页。
- 自动化读取公开网页内容。
- 对网页进行信息抓取、解析、截图、测试或数据采集。

用户可能会访问第三方网站，第三方网站可能涉及 robots.txt、服务条款、反爬策略、登录权限、数据版权、数据库权益、个人信息保护或其他合规要求。

## 三、我们希望禁止的使用行为

我们不希望也不允许用户利用代理能力从事以下行为：

- 绕过网站访问控制、验证码、登录权限、付费墙或反爬限制。
- 大规模、高频、攻击性抓取。
- 抓取、处理、买卖个人信息或敏感信息。
- 侵犯第三方知识产权、商业秘密、数据库权益或网站服务条款。
- 违法访问境外网站或规避网络监管。
- 对第三方网站造成攻击、干扰、过载或服务不可用。
- 其他违反法律法规、监管要求或第三方平台规则的行为。

## 四、希望律师评估的问题

### 4.1 代理服务本身的合规风险

在上述技术限制和产品定位下，我们向用户提供“按地域受限的代理访问能力”，是否存在以下合规风险：

- 电信业务或增值电信业务相关风险。
- 网络接入服务、代理服务、VPN 或类似服务相关监管风险。
- 网络安全、数据安全或跨境数据相关风险。
- 中国大陆、香港或其他相关地域的监管要求。

请律师判断：该服务模式是否需要取得特定资质、备案、许可或牌照。

### 4.2 用户抓取网页信息时的平台责任

用户通过我们的浏览器云平台和代理服务抓取公开网页信息时，我们作为平台或工具提供方，可能承担哪些法律责任？

请重点评估以下情形：

- 用户违反第三方网站服务条款或 robots.txt。
- 用户绕过反爬、验证码、登录权限、付费墙或访问控制。
- 用户抓取个人信息、敏感信息、商业数据、版权内容、数据库内容或其他受保护数据。
- 用户高频访问导致第三方网站损失、投诉或索赔。
- 用户将抓取结果用于违法、侵权、不正当竞争或其他违规用途。

### 4.3 产品和风控措施

我们是否可以通过产品设计、技术限制、日志留存、风控审核、账号封禁、限流策略等方式降低平台责任？

希望律师建议：

- 哪些措施是必须做的。
- 哪些措施属于强烈建议。
- 哪些措施属于加分项。

可评估的措施包括但不限于：

- 地域出口限制。
- 禁止翻墙能力。
- 访问频率限制。
- 异常访问检测。
- 账号封禁和暂停服务。
- 投诉处理机制。
- 日志留存和证据保全。
- 用户身份识别或 KYC。
- 企业客户用途审核。

### 4.4 用户条款和协议

我们需要在用户协议、服务条款、可接受使用政策、隐私政策或单独的代理服务条款中加入哪些内容，才能尽量降低法律风险？

希望律师重点覆盖以下条款方向：

- 用户必须保证其访问和抓取行为合法合规。
- 用户不得将服务用于翻墙、规避监管、攻击、侵权、非法爬取、非法处理个人信息等用途。
- 用户应自行确认目标网站的服务条款、robots.txt、授权范围和数据使用权限。
- 用户对其输入的网址、访问对象、访问行为、抓取内容、处理方式和使用结果承担责任。
- 用户不得绕过目标网站的身份验证、验证码、访问限制、反爬策略、付费墙或其他技术保护措施。
- 用户不得抓取、处理或传播个人信息、敏感信息、商业秘密、版权内容或其他受保护数据，除非其已经取得合法授权并满足适用法律要求。
- 我们有权监控异常使用、限流、暂停、终止服务，并配合监管机关或权利人投诉处理。
- 如用户违法、违规或侵权，应由用户承担责任，并赔偿我们因此产生的损失。

请律师判断这些条款是否足够，以及是否需要给出可直接放入协议的条款草案。

### 4.5 产品页面和 API 文档提示

我们是否需要在产品页面、控制台、API 文档或开发者文档中增加显著提示？

例如：

- 仅允许访问和采集合规授权的数据。
- 不得绕过访问控制、验证码、登录权限或反爬措施。
- 不得用于翻墙、违法代理访问或规避网络监管。
- 使用者应自行确认目标网站规则和数据使用权限。

请律师建议这些提示应如何表述，放在哪些位置，是否需要用户显式勾选确认。

### 4.6 企业用户和个人用户的差异

如果客户是企业客户，是否建议在合同中增加更强的承诺、保证、赔偿、审计和配合义务？

如果客户是个人开发者用户，是否需要额外限制，例如：

- 实名认证。
- 手机号或邮箱验证。
- 用途声明。
- 访问量限制。
- 更严格的风控和封禁策略。

### 4.7 日志留存和数据合规

对于日志留存和风控，我们应该保留哪些信息、保留多久，既能满足风险处置和举证需要，又不引入过度收集个人信息或数据合规风险？

希望律师评估：

- 是否应记录用户账号、访问时间、目标域名、代理地域、请求量、异常行为等信息。
- 是否应避免记录完整网页内容、敏感参数或个人信息。
- 日志保留周期建议。
- 向用户披露日志和风控处理的方式。

## 五、希望律师输出的结果

希望律师最终输出：

1. 该代理服务模式的主要法律风险清单。
2. 是否需要资质、备案、许可或牌照的判断。
3. 推荐的产品合规控制措施。
4. 推荐的用户协议、可接受使用政策、代理服务条款中的关键条款。
5. 如有必要，请提供适合直接放入协议的条款草案。
6. 对企业客户和个人开发者用户分别适用的合规建议。
7. 对日志留存、风控、KYC 或用户身份识别的建议。

## 六、建议提供给律师的技术材料

为便于律师判断，我们可以同步准备以下材料：

- 第三方代理供应商的账号体系和地域配置说明。
- 国内 IP 池、香港/海外 IP 池的出口隔离规则。
- 禁止通过代理翻墙的产品配置说明。
- 用户使用代理服务的产品流程。
- API 文档或开发者文档中与代理相关的说明。
- 当前访问日志字段。
- 当前风控、限流、封禁和投诉处理机制。
- 用户协议、隐私政策、可接受使用政策的现有版本。
