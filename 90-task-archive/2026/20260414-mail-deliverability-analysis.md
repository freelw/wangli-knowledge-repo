# 2026-04-14 lex-home 邮件送达率治理分析

## 背景

`lex-home` 存在邮件经常进入垃圾箱的问题，尤其影响注册验证邮件、重置密码邮件和联系表单通知这类高价值事务邮件。

## 任务目标

1. 判断当前邮件进入垃圾箱的根因
2. 给出短期止损方案
3. 给出长期可治理的邮件发送方案

## 涉及仓库

1. `lex-home`

## 关键改动

这次任务的核心产出是方案分析，而不是代码变更。

关键结论包括：

1. 当前问题大概率不在模板，而在发信基础设施、域名认证和信誉管理
2. 事务邮件不适合继续长期依赖低信誉 SMTP 链路
3. 现有代码已内置 `resend` provider，接入成本低
4. 建议采用“两阶段治理”：
   - 先诊断当前 SMTP 链路
   - 再把关键事务邮件迁移到 Resend 类成熟服务

## 验证方式

这次任务本身属于分析和方案制定，未形成代码发布验证。

建议的后续验证方式包括：

1. 抓取真实样本邮件头
2. 核对 SPF / DKIM / DMARC / PTR
3. 小规模实投 Gmail / Outlook / 企业邮箱
4. 迁移后观察 inbox rate 和投诉 / bounce 情况

## 结论

1. 当前邮件问题的主要矛盾在发信信誉体系，而不是模板本身
2. `Resend` 是当前最低摩擦的事务邮件迁移方向
3. 更合理的长期方案是事务邮件与未来营销邮件使用分离域名
4. 这次任务为后续邮件治理建立了明确的诊断和迁移框架
